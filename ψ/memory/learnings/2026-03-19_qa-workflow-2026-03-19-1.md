# ## บทเรียน QA Workflow จากวันที่ 2026-03-19

## บทเรียน QA Workflow จากวันที่ 2026-03-19

### 1. หา thread ที่ถูกต้อง — ลำดับ 3 ขั้น
- ขั้น 1: `oracle_threads()` → ดู title ทั้งหมด → หา `channel:<oracle-name>`
- ขั้น 2: ถ้าไม่เจอ → ข้อมูลอยู่ใน notification text ที่ส่งมาทาง tmux ใช้ได้เลย
- ขั้น 3: `maw talk-to <oracle>` ถามตรง
- ❌ ห้ามวน oracle_thread_read ID ที่ผิดซ้ำๆ

### 2. QA Checklist มาตรฐาน — ใช้ทุกครั้งที่รีวิวเว็บ
1. เปิดได้ไหม
2. Content ครบไหม
3. Responsive (mobile/tablet/desktop)
4. Accessibility (skip-link, aria, keyboard, focus, screen reader)
5. Styling/Theme ถูกต้อง
6. OG tags (og:title, og:description, og:type)
7. Links ถูกต้อง ไม่ชี้ผิด

### 3. WebFetch URL จริงก่อนตอบเสมอ
- ตามกฎข้อ 7: ตรวจข้อมูลจริงก่อนทำงานเสมอ
- ห้ามอ่านแค่โค้ดแล้วตอบ — ต้อง WebFetch ดูเว็บจริงด้วย

### 4. รีวิว 2 รอบ flow
- รอบ 1: รีวิวหลัก → feedback ชัดเจน จุดไหน critical/important → `maw talk-to <oracle>`
- รอบ 2: เช็คว่าแก้ครบทุกจุด → APPROVED → `maw done`
- ส่ง inbox signal ให้ทั้ง Oracle ที่ทำงาน + แม่ (xToriMicz) ทุกรอบ

### 5. Token/Context Management
- 50%+ → ห้ามเริ่มงานใหม่
- 60%+ → ให้ `maw done` สรุปงาน เขียน handoff
