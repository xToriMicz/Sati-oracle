# ## กฎเหล็ก 4 ข้อสำหรับการทำงานในระบบ Oracle

## กฎเหล็ก 4 ข้อสำหรับการทำงานในระบบ Oracle

### 1. ใช้ maw ไม่ใช่ npx maw
- ❌ `npx maw talk-to` → error เพราะหา src/ ไม่เจอ
- ✅ `maw talk-to jingjing-oracle "ข้อความ"` → ใช้ global binary ตรงเลย
- maw ถูก install เป็น global binary แล้ว ไม่ต้อง npx

### 2. ชื่อ Oracle ต้องมี -oracle ต่อท้ายเสมอ
- ❌ `maw talk-to jingjing`
- ✅ `maw talk-to jingjing-oracle`
- tmux window ชื่อ Jingjing-oracle, Sati-oracle → ต้อง match กัน

### 3. หา thread จาก Oracle อื่น — อย่าเชื่อ ID ตรงๆ
- notification อาจบอก thread ID ผิด
- ✅ ใช้ oracle_threads() ดู title ทั้งหมด → หา channel ที่ตรง เช่น "channel:jingjing-oracle"
- ถ้าหาไม่เจอใน thread → ข้อมูลอยู่ใน notification text เลย ใช้ได้ทันที
- ❌ ห้ามวนหา thread ซ้ำๆ 5 รอบโดยไม่ลองวิธีอื่น

### 4. ส่ง maw talk-to แล้วห้าม idle
- ❌ "รอจิงจิงส่งมาครับ" แล้วหยุด
- ✅ poll oracle_thread_read ทุก 30 วิ อย่างน้อย 5 รอบ
- ต้อง active poll ไม่ใช่ passive wait
